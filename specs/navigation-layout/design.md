## CSS variables in globals.css (shadcn convention)

shadcn uses `--background`, `--foreground`, `--accent`, `--muted`, `--border`, etc.
Map the existing JobScout design tokens onto these names:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  :root {                          /* dark default */
    --background: 222 47% 7%;
    --foreground: 215 28% 89%;
    --surface: 225 23% 14%;
    --accent: 239 84% 67%;
    --muted: 215 16% 57%;
    --border: 226 20% 22%;
    --sidebar-width: 220px;
    --font-sans: "Inter", sans-serif;
    --font-mono: "JetBrains Mono", monospace;
  }
  .light {                         /* light theme class (shadcn uses "light", not "theme-light") */
    --background: 210 40% 98%;
    --foreground: 222 47% 7%;
    --surface: 0 0% 100%;
    --accent: 243 75% 59%;
    --muted: 215 16% 47%;
    --border: 214 32% 91%;
  }
}
```

**Theme class:** shadcn's convention uses `"light"` / `"dark"` on `<html>`, not `"theme-light"` / `"theme-dark"`. Update `useTheme.ts` and R7/R8/R9 accordingly.

| Channel | Direction | Used by |
|---|---|---|
| `profiles:list` | renderer → main | ProfileSwitcher (on mount) |

These channels are expected to be registered by the `profiles` feature. If they
are absent the ProfileSwitcher MUST handle the rejection gracefully (show an
error state, do not crash).

## Discarded alternative: TanStack Router

TanStack Router v1 provides full type-safety for route params and search params.
Discarded because: (1) the project already lists `react-router-dom` as the
dependency and reversing that decision requires a feature-level discussion;
(2) type-safe route params are not needed for this shell — all routes are static
paths; (3) introduces an additional dependency without concrete benefit at this
stage, violating the "minimal dependencies" principle in `docs/architecture.md`.

## Discarded alternative: Styled Components for theming

Using a CSS-in-JS library (Styled Components or Emotion) would simplify dynamic
theming via props. Discarded because `docs/conventions.md` and the feature brief
explicitly require CSS Modules + CSS variables. The `theme-dark` / `theme-light`
class-on-`<html>` pattern achieves the same result with zero extra runtime.
