- TypeScript strict mode, no `any`
- Tailwind CSS + shadcn/ui only — no CSS Modules, no inline styles, no CSS-in-JS
- Use createHashRouter (required for Electron file:// in prod)
- Theme class is "dark" / "light" on <html> (shadcn convention)
- Theme persisted in localStorage key "theme", default is "dark"

- ProfileSwitcher uses shadcn DropdownMenu and calls:
    window.api.invoke('profiles:list')

- Each route renders a placeholder <div> only — no real screen content yet
- After all tasks done, run: pnpm test && npx tsc --noEmit

## ⬜ 7. scraping-engine
Playwright pipeline (main process):
1. Navigate to board URL
2. Find search bar via heuristic cascade (6 selectors) or saved CSS selector
3. If not found: pause, show screenshot, ask user for selector
4. Type keyword → submit → wait for networkidle
5. Scrape job cards (title, company, location, date, snippet, URL)
6. Paginate while jobs are within date range cutoff
7. Deduplicate by sanitized URL → insert new jobs with status `new`

Depends on: boards-management ✅, keywords-management ✅

**Before implementing:** ask Claude Code to generate the spec.
> "generate spec for scraping-engine"

## ⬜ 8. ai-matching
Three-phase AI pipeline run after each scout session:
- Phase 1: Resume skill profile — 1 Ollama/Anthropic call, cached until resume changes
- Phase 2: Batch scoring — 5 jobs per call, returns score 0–100
- Phase 3: Match reason — 1 call per job scoring ≥70, capped at 10 calls/session

Supports two backends: Ollama (default, local, free) + Anthropic Claude API (optional).

Depends on: scraping-engine ✅, resume-upload ✅

**Before implementing:** ask Claude Code to generate the spec.
> "generate spec for ai-matching"
